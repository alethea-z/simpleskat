# Runbook

## Start order
1. Read the project idea.
2. Load the current `STATE.md`.
3. Load the operating model, state machine, requirements conversation, transparency, and quality gate notes.
4. Run the requirements conversation.
5. Write the discovery result.
6. Turn the discovery result into testable requirements.
7. Check readiness for planning.
8. Create the first small issue.
9. Implement the issue.
10. Verify the result.
11. Show it to the human and capture feedback.

## Working rule
If a step becomes unclear, stop and record the blocker before continuing.
